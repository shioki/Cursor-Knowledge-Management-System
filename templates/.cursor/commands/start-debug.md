# デバッグセッション開始

問題の調査を開始し、デバッグセッションを記録します。

## 手順

1. ユーザーに以下の情報をヒアリングしてください:
   - **問題の概要**: 何が起きているか
   - **環境**: 開発/ステージング/本番
   - **再現手順**: どのようにすれば再現できるか
   - **エラーメッセージ**: 表示されているエラー（あれば）
   - **期待動作**: 本来どう動くべきか
   - **緊急度**: 高/中/低

2. `.claude/skills/debug-workflow/references/DEBUG_TEMPLATE.md`（または `.cursor/skills/...`）を読み込み、過去の類似問題を確認してください。

3. `.claude/debug-sessions/`（または `.cursor/debug-sessions/`）ディレクトリがあれば、過去のセッションで類似問題がないか確認してください。

4. デバッグセッションファイルを `.claude/debug-sessions/YYYY-MM-DD_問題概要.md`（または `.cursor/debug-sessions/...`）として作成してください:

```markdown
# デバッグセッション: [問題の概要]

## 基本情報
| 項目 | 内容 |
|------|------|
| 発生日時 | YYYY-MM-DD HH:MM |
| 環境 | [環境] |
| 影響範囲 | [範囲] |
| 緊急度 | [高/中/低] |
| ステータス | 調査中 |

## 症状
### エラーメッセージ
### 再現手順
### 期待動作
### 実際の動作

## 調査
### 仮説
### 調査ログ

## 解決策
### 根本原因
### 修正内容
### テスト結果

## 再発防止
```

5. 問題の仮説を立て、調査方針を提案してください。

6. 解決後は `/log-improvement` コマンドで再発防止策を記録するよう案内してください。

## 注意事項

- 調査の各ステップを記録してください（後で類似問題の参考になります）
- 緊急度が高い場合は暫定対応を優先してください
- 解決後は必ず根本原因を記録してください
