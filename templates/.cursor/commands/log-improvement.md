# 改善を記録

コード品質の改善、リファクタリング、最適化の内容を記録します。

## 手順

1. ユーザーに以下の情報をヒアリングしてください:
   - **改善タイトル**: 簡潔な改善内容の説明
   - **背景**: 改善の動機・発見の経緯
   - **改善内容**: 具体的に何をしたか／するか
   - **効果**: Before/After の比較（可能であれば数値で）
   - **ステータス**: 提案/実装中/完了/保留

2. `.cursor/skills/improvement-tracking/references/IMPROVEMENTS_TEMPLATE.md` を読み込み、既存の改善記録を確認してください。

3. 以下のテンプレートでエントリを作成し、適切なセクションに追記してください:

```markdown
### YYYY-MM-DD - 改善タイトル

#### 背景
改善の動機・発見の経緯

#### 改善内容
具体的な改善策

#### 効果
- **Before**:
- **After**:
- **改善率**:

#### ステータス
提案 / 実装中 / 完了 / 保留
```

4. 関連する技術判断やパターンがあれば、紐付けを提案してください。

## 注意事項

- 効果は可能な限り数値で記録してください（レスポンス時間、コード行数、テストカバレッジ等）
- デバッグセッションから派生した改善は、セッションへのリンクを含めてください
- 技術的負債の解消は「技術的負債解消」セクションに記録してください
